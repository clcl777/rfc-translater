A Uniform Resource Name (URN) Namespace for the Hybrid Broadcast Broadband TV (HbbTV) Association

ハイブリッドブロードキャストブロードバンドTV（HbbTV）協会のUniform Resource Name（URN）名前空間

Abstract

概要

This document describes a Uniform Resource Name (URN) namespace for the Hybrid Broadcast Broadband TV (HbbTV) Association for naming persistent resources defined within HbbTV specifications. Example resources include technical documents and specifications, Extensible Markup Language (XML) Schemas, classification schemes, XML Document Type Definitions (DTDs), namespaces, style sheets, media assets, and other types of resources produced or managed by HbbTV.

このドキュメントでは、HbbTV仕様内で定義された永続リソースに名前を付けるためのハイブリッドブロードキャストブロードバンドTV（HbbTV）協会のUniform Resource Name（URN）名前空間について説明します。リソースの例には、技術ドキュメントと仕様、Extensible Markup Language（XML）スキーマ、分類スキーマ、XMLドキュメントタイプ定義（DTD）、名前空間、スタイルシート、メディアアセット、およびHbbTVによって作成または管理されるその他のタイプのリソースが含まれます。

1. Introduction
1. はじめに

HbbTV (Hybrid Broadcast Broadband TV) is a new industry standard providing an open and business-neutral technology platform that seamlessly combines TV services delivered via broadcast with services delivered via broadband and also enables access to Internet-only services for consumers using connected TVs and set-top boxes. The founding members of the HbbTV consortium together with a large group of supporters jointly developed the HbbTV specification to create a global standard for hybrid entertainment services. Version 1.2.1 of this specification was published by ETSI as ETSI TS 102 796 [HbbTV12] in November 2012.

HbbTV（Hybrid Broadcast Broadband TV）は、放送を介して配信されるTVサービスとブロードバンドを介して配信されるサービスをシームレスに組み合わせ、接続されたTVとセットを使用する消費者向けのインターネットのみのサービスへのアクセスを可能にするオープンでビジネス中立のテクノロジープラットフォームを提供する新しい業界標準です。トップボックス。 HbbTVコンソーシアムの創設メンバーは、大勢の支持者グループと共同でHbbTV仕様を開発し、ハイブリッドエンターテインメントサービスのグローバル標準を作成しました。この仕様のバージョン1.2.1は、ETSIによって2012年11月にETSI TS 102 796 [HbbTV12]として公開されました。

The HbbTV specification is based on existing standards and web technologies including the Open IPTV Forum (OIPF), Consumer Electronics Association (CEA), Digital Video Broadcasting (DVB), and World Wide Web Consortium (W3C). The standard provides the features and functionality required to deliver feature-rich broadcast and Internet services. Utilizing standard Internet technology, it enables rapid application development. It defines minimum requirements simplifying the implementation in devices and leaving room for differentiation; this limits the investment required by consumer electronics manufacturers to build compliant devices. The HbbTV Association is a pan-European initiative aimed at providing an alternative to proprietary technologies and delivering an open platform for broadcasters to deliver value-added on-demand services to the end consumer.

HbbTV仕様は、Open IPTV Forum（OIPF）、Consumer Electronics Association（CEA）、Digital Video Broadcasting（DVB）、World Wide Web Consortium（W3C）などの既存の標準とWebテクノロジに基づいています。この標準は、機能豊富なブロードキャストおよびインターネットサービスを提供するために必要な機能を提供します。標準のインターネット技術を利用して、迅速なアプリケーション開発を可能にします。デバイスの実装を簡素化し、差別化の余地を残す最小要件を定義します。これにより、家電メーカーが準拠デバイスを構築するために必要な投資が制限されます。 HbbTVアソシエーションは、独自技術に代わるものを提供し、放送局が付加価値のあるオンデマンドサービスをエンドコンシューマーに提供するためのオープンプラットフォームを提供することを目的とした汎ヨーロッパイニシアチブです。

HbbTV has a wide range of supporters from across the broadcaster and consumer electronics industries.

HbbTVには、放送業界および家電業界の幅広いサポーターがいます。

HbbTV would like to assign unique, permanent, location-independent names based on URNs for some resources it produces or manages. These URNs will be constructed according to the URN syntax defined in [RFC2141]. This namespace specification is for a formal namespace to be registered according to the procedures set forth in [RFC3406].

HbbTVは、生成または管理する一部のリソースに対して、URNに基づいて、場所に依存しない一意の永続的な名前を割り当てたいと考えています。これらのURNは、[RFC2141]で定義されているURN構文に従って構築されます。この名前空間の仕様は、[RFC3406]で規定されている手順に従って登録される正式な名前空間用です。

4. Namespace Considerations
4. 名前空間に関する考慮事項

A unique formal namespace is required by HbbTV in order to specify how the various existing standards can be linked in order to create a true end-to-end ecosystem for standards-based IPTV deployments and to provide the necessary system-wide resources.

標準ベースのIPTV展開のための真のエンドツーエンドのエコシステムを作成し、必要なシステム全体のリソースを提供するために、さまざまな既存の標準をリンクする方法を指定するために、HbbTVには一意の正式な名前空間が必要です。

URN assignment procedures:

URN割り当て手順：

The individual URNs shall be assigned through the process of development of HbbTV specifications by the Hybrid Broadcast Broadband TV Association. The latest information about HbbTV-defined specifications can always be found at the owner's website at

個々のURNは、Hybrid Broadcast Broadband TV AssociationによるHbbTV仕様の開発プロセスを通じて割り当てられます。 HbbTV定義の仕様に関する最新情報は、常に次の所有者のWebサイトにあります。 

         <https://hbbtv.org/pages/about_hbbtv/specification.php>

URN resolution/delegation:

URN解決/委任：

The resolution and delegation shall be determined through the process of development of specifications by the HbbTV Association.

解決と委任は、HbbTV協会による仕様の開発プロセスを通じて決定されるものとします。

Since the implementations envisaged cover a wide range of devices with quite different access methods and capabilities, no single resolution or delegation mechanism can be referenced in this document.

想定されている実装は、まったく異なるアクセス方法と機能を備えた幅広いデバイスを対象としているため、このドキュメントでは単一の解決または委任メカニズムを参照することはできません。

Type of resources to be identified:

識別されるリソースのタイプ：

Types of resources to be identified include XML schema definition files, classification schemes, and identification systems defined and published by HbbTV. These resources being identified constitute a metadata system to describe digital multimedia broadcast services or content conveyed as part of such services.

識別されるリソースのタイプには、XMLスキーマ定義ファイル、分類スキーマ、およびHbbTVによって定義および公開された識別システムが含まれます。識別されるこれらのリソースは、デジタルマルチメディアブロードキャストサービスまたはそのようなサービスの一部として伝達されるコンテンツを記述するメタデータシステムを構成します。

The latest HbbTV-defined specifications can always be found at

最新のHbbTV定義の仕様は常に次の場所にあります。 

         <https://hbbtv.org/pages/about_hbbtv/specification.php>
5. Community Considerations
5. コミュニティの考慮事項

URNs defined by HbbTV will be used by implementers of hybrid broadcast/broadband systems, services, products, and applications based on the HbbTV specification. They are an essential component of the open ecosystem that is being facilitated by HbbTV.

HbbTVによって定義されたURNは、HbbTV仕様に基づくハイブリッドブロードキャスト/ブロードバンドシステム、サービス、製品、およびアプリケーションの実装者によって使用されます。これらは、HbbTVによって促進されているオープンエコシステムの重要なコンポーネントです。

HbbTV specifications are developed through a consensus-based process by member organizations representing most aspects of the digital television ecosystem. A full list of HbbTV members can be found at

HbbTV仕様は、デジタルテレビエコシステムのほとんどの側面を代表するメンバー組織によって、コンセンサスベースのプロセスを通じて開発されています。 HbbTVメンバーの完全なリストは、次の場所にあります。 

      <http://hbbtv.org/pages/hbbtv_association/members.php>
6. Security Considerations
6. セキュリティに関する考慮事項

There are no additional security considerations other than those normally associated with the use and resolution of URNs in general, which are described in [RFC1737], [RFC2141], and [RFC3406].

[RFC1737]、[RFC2141]、および[RFC3406]で説明されている、URNの一般的な使用と解決に通常関連するセキュリティ以外の考慮事項はありません。

This document registers a namespace for URNs. HbbTV may assign special meaning to certain of the characters of the Namespace Specific String (NSS) in its specifications. Any security consideration resulting from such assignment is outside the scope of this document.

このドキュメントでは、URNの名前空間を登録します。 HbbTVは、その仕様で名前空間固有文字列（NSS）の特定の文字に特別な意味を割り当てる場合があります。このような割り当ての結果として生じるセキュリティ上の考慮事項は、このドキュメントの範囲外です。
