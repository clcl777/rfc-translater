1. Introduction
1. はじめに

This document presents three IETF-defined YANG modules [RFC7950] and a script used to create four supporting IANA modules.

このドキュメントでは、3つのIETF定義のYangモジュール[RFC7950]と、4つのサポートするIANAモジュールを作成するために使用されるスクリプトを提示します。

The three IETF modules are ietf-ssh-common (Section 2), ietf-ssh-client (Section 3), and ietf-ssh-server (Section 4). The "ietf-ssh-client" and "ietf-ssh-server" modules are the primary productions of this work, supporting the configuration and monitoring of SSH clients and servers.

3つのIETFモジュールは、IETF-SSH-Common（セクション2）、IETF-SSH-Client（セクション3）、およびIETF-SSH-Server（セクション4）です。「IETF-SSH-Client」および「IETF-SSH-Server」モジュールは、この作業の主要な作品であり、SSHクライアントとサーバーの構成と監視をサポートしています。

The groupings defined in this document are expected to be used in conjunction with the groupings defined in an underlying transport-level module, such as the groupings defined in [RFC9643]. The transport-level data model enables the configuration of transport-level values, such as a remote address, a remote port, a local address, and a local port.

このドキュメントで定義されているグループは、[RFC9643]で定義されたグループ化など、基礎となる輸送レベルモジュールで定義されたグループ化と組み合わせて使用されることが期待されています。トランスポートレベルのデータモデルにより、リモートアドレス、リモートポート、ローカルアドレス、ローカルポートなど、トランスポートレベルの値の構成が可能になります。

1.1. Regarding the Three IETF Modules
1.1. 3つのIETFモジュールについて

The three IETF modules define features and groupings to model "generic" SSH clients and SSH servers, where "generic" should be interpreted as "least common denominator" rather than "complete." Support for the basic SSH protocol [RFC4252] [RFC4253] [RFC4254] is afforded by these modules, leaving configuration of advanced features (e.g., multiple channels) to augmentations made by consuming modules.

3つのIETFモジュールは、「汎用」SSHクライアントとSSHサーバーをモデル化する機能とグループ化を定義します。ここで、「汎用」は「完全」ではなく「最も一般的な分母」と解釈されるべきです。基本的なSSHプロトコル[RFC4252] [RFC4253] [RFC4254]のサポートは、これらのモジュールによって提供され、モジュールを消費することで作られた増強に高度な機能（複数のチャネル）の構成を残します。

It is intended that the YANG groupings will be used by applications needing to configure SSH client and server protocol stacks. For instance, these groupings are used to help define the data models in [NETCONF-CLIENT-SERVER], for clients and servers using the Network Configuration Protocol (NETCONF) over SSH [RFC6242].

Yangグループは、SSHクライアントとサーバーのプロトコルスタックを構成する必要があるアプリケーションで使用されることを意図しています。たとえば、これらのグループは、SSH [RFC6242]を介してネットワーク構成プロトコル（NetConf）を使用してクライアントとサーバーの[NetConf-Client-Server]のデータモデルを定義するのに役立ちます。

The "ietf-ssh-client" and "ietf-ssh-server" YANG modules each define one grouping, which is focused on just SSH-specific configuration, and specifically avoid any transport-level configuration, such as what ports to listen on or connect to. This affords applications the opportunity to define their own strategy for how the underlying TCP connection is established. For instance, applications supporting NETCONF Call Home [RFC8071] could use the "ssh-server-grouping" grouping for the SSH parts it provides while adding data nodes for the TCP-level call-home configuration.

「IETF-SSH-Client」および「IETF-SSH-Server」Yangモジュールはそれぞれ1つのグループを定義します。に接続します。これにより、アプリケーションは、基礎となるTCP接続がどのように確立されるかについての独自の戦略を定義する機会を提供します。たとえば、NetConf Call Home [RFC8071]をサポートするアプリケーションは、TCPレベルのコールホーム構成のデータノードを追加しながら、提供するSSH部品の「SSH-Server-Grouping」グループを使用できます。

The modules defined in this document optionally support [RFC6187], which describes enabling host keys and public keys based on X.509v3 certificates.

このドキュメントで定義されているモジュールは、オプションで[RFC6187]をサポートします。これは、X.509V3証明書に基づいてホストキーとパブリックキーを有効にすることを説明しています。

2. The "ietf-ssh-common" Module
2. 「IETF-SSH-Common」モジュール

The SSH common model presented in this section is common to both SSH clients and SSH servers. The "transport-params-grouping" grouping can be used to configure the list of SSH transport algorithms permitted by the SSH client or SSH server. The lists of permitted algorithms are in decreasing order of usage preference. The algorithm that appears first in the client list that also appears in the server list is the one that is used for the SSH transport layer connection. The ability to restrict the algorithms allowed is provided in this grouping for SSH clients and SSH servers that are capable of doing so and may serve to make SSH clients and SSH servers compliant with security policies.

このセクションで紹介するSSH共通モデルは、SSHクライアントとSSHサーバーの両方に共通しています。「Transport-Params-Grouping」グループ化を使用して、SSHクライアントまたはSSHサーバーが許可するSSHトランスポートアルゴリズムのリストを構成できます。許可されているアルゴリズムのリストは、使用法の好みの順序を減らしています。サーバーリストにも表示されるクライアントリストに最初に表示されるアルゴリズムは、SSH輸送層接続に使用されるものです。許可されているアルゴリズムを制限する機能は、SSHクライアントとSSHサーバーのこのグループで提供され、SSHクライアントとSSHサーバーをセキュリティポリシーに準拠させるのに役立つ場合があります。

4.1.3. Protocol-Accessible Nodes
4.1.3. プロトコルアクセス可能なノード

The "ietf-ssh-server" module defines only "grouping" statements that are used by other modules to instantiate protocol-accessible nodes. Thus, this module, when implemented, does not itself define any protocol-accessible nodes.

「IETF-SSH-SERVER」モジュールは、他のモジュールによって使用される「グループ化」ステートメントのみを定義し、プロトコルアクセス可能なノードをインスタンス化します。したがって、このモジュールは、実装された場合、プロトコルアクセス可能なノードを定義するものではありません。
