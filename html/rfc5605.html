Abstract

概要

This memo defines a portion of the Management Information Base (MIB) for use with network management protocols in the Internet community. In particular, it describes managed objects for modeling ATM Pseudowire (PW) carrying ATM cells over Packet Switched Networks (PSNs).

このメモは、インターネットコミュニティのネットワーク管理プロトコルで使用するための管理情報ベース（MIB）の一部を定義します。特に、Packet Switched Network（PSNS）を介してATMセルを運ぶATM pseudowire（PW）をモデル化するための管理オブジェクトについて説明します。

1. Introduction
1. はじめに

This document describes a model for managing "emulated" ATM services over a Packet Switched Network (PSN).

このドキュメントでは、パケットスイッチネットワーク（PSN）を介して「エミュレート」ATMサービスを管理するためのモデルについて説明します。

The document follows the requirements for Pseudowire Emulation Edge-to-Edge [PWREQ]; it is closely related to [ATMENCAP] and [ATMTRANS], which describe the encapsulation of ATM signals and provide the Emulation Service over a Packet Switched Network.

このドキュメントは、Pseudowire Emulation Edge-to-Edge [PWREQ]の要件に従います。これは、[Atmencap]および[atmtrans]に密接に関連しており、ATM信号のカプセル化を記述し、パケットスイッチネットワークを介してエミュレーションサービスを提供します。

The ATM management model consists of several MIB modules, following the layering model described in the PWE3 Architecture [PWARCH] document. The ATM MIB module described in this document works closely with the MIB modules described in [AToMTC], [AToM], [IFMIB], [PWMIB], and the textual conventions defined in [PWTC]. The conceptual layering and relationship among all of those is described in Figure 1 and in the "Relation to Other PW-MIB Modules" section listed below. An ATM connection will be a pseudowire (PW) connection. It will not be treated as an interface and will therefore not be represented in the ifTable.

ATM管理モデルは、PWE3アーキテクチャ[PWARCH]ドキュメントに記載されている階層モデルに従って、いくつかのMIBモジュールで構成されています。このドキュメントで説明されているATM MIBモジュールは、[ATOMTC]、[ATOM]、[IFMIB]、[PWMIB]、および[PWTC]で定義されたテキスト規則で説明されているMIBモジュールと密接に連携します。これらすべての概念的な階層化と関係は、図1と「他のPW-MIBモジュールとの関係」セクションで説明されています。ATM接続は、擬似ワイヤ（PW）接続になります。それはインターフェイスとして扱われないため、IFTableで表されません。

3. Terminology
3. 用語

This document follows the terminology used in PW Architecture [PWARCH].

このドキュメントは、PWアーキテクチャ[PWARCH]で使用される用語に従います。

PSN-bound References the traffic direction where an ATM Cell is received, adapted to the packet, assigned a PW label, and sent into the PSN. Within the MIB objects, it is called outbound.

PSNバウンドは、ATMセルが受信され、パケットに適応され、PWラベルを割り当て、PSNに送信される交通方向を参照します。MIBオブジェクト内では、アウトバウンドと呼ばれます。

CE-bound The direction where packets are received from the PSN, cells are reconstructed from the packet payloads, and are sent into the ATM network as cells. Within the MIB objects, it is called inbound.

CEバウンドPSNからパケットが受信される方向、セルはパケットペイロードから再構築され、セルとしてATMネットワークに送信されます。MIBオブジェクト内では、インバウンドと呼ばれます。

Adaptation Refers to the method of adapting a "foreign" communications protocol such that it can be carried by a packet switched net (the PSN). For example, in an ATM service, the foreign protocol is ATM. The PSN may be MPLS.

適応とは、「外国の」通信プロトコルを適応させる方法を指し、パケットスイッチ付きネット（PSN）で運ぶことができます。たとえば、ATMサービスでは、外国プロトコルはATMです。PSNはMPLSである場合があります。

PSN Packet Switched Network.

PSNパケットスイッチネットワーク。

PSN Tunnel A general term indicating a virtual connection between the two PW edge devices. In practice, this connection is not limited to path-oriented types of PSNs such as MPLS. An example of a non-path-oriented PSN is an IP PSN.

PSNトンネル2つのPWエッジデバイス間の仮想接続を示す一般用語。実際には、この接続は、MPLSなどのパス指向のPSNSに限定されません。非パス指向PSNの例は、IP PSNです。

6. Relation to Other PW-MIB Modules
6. 他のPW-MIBモジュールとの関係

The MIB structure for defining a PW service is composed of three layers of MIB modules functioning together. This general model is defined in the PWE3 Architecture [PWARCH]. The layering model is intended to sufficiently isolate PW services from the underlying PSN layer that carries the emulated service. This is done at the same time as providing a standard means for connecting any supported services to any supported PSNs.

PWサービスを定義するためのMIB構造は、一緒に機能する3層のMIBモジュールで構成されています。この一般的なモデルは、PWE3アーキテクチャ[PWARCH]で定義されています。層状モデルは、エミュレートされたサービスを運ぶ基礎となるPSN層からPWサービスを十分に分離することを目的としています。これは、サポートされているサービスをサポートされているPSNSに接続するための標準手段を提供すると同時に行われます。

The first layer, known as the service layer, contains service-specific modules such as the one defined in this document. These modules define service-specific management objects that interface or collaborate with existing MIB modules for the native version of the service. The service-specific module "glues" the standard module to the PWE MIB framework.

サービスレイヤーと呼ばれる最初のレイヤーには、このドキュメントで定義されているようなサービス固有のモジュールが含まれています。これらのモジュールは、サービスのネイティブバージョンの既存のMIBモジュールとインターフェイスまたはコラボレーションするサービス固有の管理オブジェクトを定義します。サービス固有のモジュールは、PWE MIBフレームワークの標準モジュールを「接着」します。

The next layer of the PWE MIB framework is comprised of the PW-MIB module [PWMIB]. This module is used to configure general parameters of PW connections that are common to all types of emulated services and PSNs. This layer is connected to the service-specific layer above, and the PSN layer below.

PWE MIBフレームワークの次の層は、PW-MIBモジュール[PWMIB]で構成されています。このモジュールは、あらゆるタイプのエミュレートサービスとPSNSに共通のPW接続の一般的なパラメーターを構成するために使用されます。このレイヤーは、上のサービス固有のレイヤーに接続され、下のPSNレイヤーに接続されています。

The PSN layer provides PSN-specific modules for each type of PSN. These modules associate the PW with one or more "tunnels" that carry the service over the PSN. These modules are defined in other documents. This module is used to "glue" the PW service to the underlying PSN-specific MIB modules. In the case of MPLS, for example, the PW-MPLS MIB [PWMPLSMIB] is used to connect the PW service to either the MPLS-LDP [LDPMIB] or MPLS-TE [TEMIB] MIBs.

PSNレイヤーは、PSNの各タイプにPSN固有のモジュールを提供します。これらのモジュールは、PWをPSNを介してサービスを運ぶ1つ以上の「トンネル」に関連付けます。これらのモジュールは、他のドキュメントで定義されています。このモジュールは、基礎となるPSN固有のMIBモジュールへのPWサービスを「接着」するために使用されます。たとえば、MPLSの場合、PW-MPLS MIB [PWMPLSMIB]を使用して、PWサービスをMPLS-LDP [LDPMIB]またはMPLS-TE [TEMIB] MIBSのいずれかに接続します。

[PWTC] defines some of the object types used in these modules.

[PWTC]これらのモジュールで使用されるオブジェクトタイプの一部を定義します。

8. Structure of the MIB Module
8. MIBモジュールの構造

This MIB consists of 4 types of tables;

このMIBは、4種類のテーブルで構成されています。

It is important to note that the TrafficParamDescr Table is not defined as part of this MIB, although an object pointing to such a table entry exists in all configuration tables of this MIB module. Users can refer to any ATM TrafficDescr (TD) Table if there is a need to overwrite the TD assigned to the ATM endpoint in the ATM service MIB [AToM].

このMIBモジュールのすべての構成テーブルには、このようなテーブルエントリを指すオブジェクトはこのMIBの一部として定義されていないことに注意することが重要です。ユーザーは、ATMサービスMIB [ATOM]のATMエンドポイントに割り当てられたTDを上書きする必要がある場合、ATM TrafficDescr（TD）テーブルを参照できます。

o PW ATM Cfg Table: A table for generic parameters for ATM PW configuration that is applicable for each ATM PW.

o PW ATM CFGテーブル：各ATM PWに適用可能なATM PW構成の一般的なパラメーターのテーブル。

o PW ATM Outbound Table: There are two tables to configure an outbound ATM PW depending on the type of service. One table for 1:1 service, and the other for N:1 service and transparent cell mode [ATMTRANS].

o PW ATMアウトバウンドテーブル：サービスの種類に応じて、アウトバウンドATM PWを構成する2つのテーブルがあります。1：1サービス用のテーブル、もう1つはn：1サービスおよび透明セルモード[ATMTrans]のテーブル。

o PW ATM Inbound Table: There are two tables to configure an inbound ATM PW depending on the type of service. One table for 1:1 service, and the other for N:1 service and transparent cell mode.

o PW ATMインバウンドテーブル：サービスの種類に応じて、インバウンドATM PWを構成する2つのテーブルがあります。1：1サービス用のテーブル、もう1つはn：1サービスと透明セルモードのテーブルです。

o PW ATM Perf Table: There are three tables; each contains the relevant time-dependent statistics for an ATM PW Entry. There is a current table, a 15-minute interval table, and a one-day interval table. The tables are aligned with statistic models of other PW services.

o PW ATMパフォーマンステーブル：3つのテーブルがあります。それぞれに、ATM PWエントリの関連する時間依存統計が含まれています。現在のテーブル、15分間のインターバルテーブル、1日のインターバルテーブルがあります。テーブルは、他のPWサービスの統計モデルと一致しています。

10. Security Considerations
10. セキュリティに関する考慮事項

There are a number of management objects defined in this MIB module with a MAX-ACCESS clause of read-write and/or read-create. Such objects may be considered sensitive or vulnerable in some network environments.
