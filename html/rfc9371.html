Registration Procedures for Private Enterprise Numbers (PENs)

民間企業番号（ペン）の登録手順

Abstract

概要

This document describes how Private Enterprise Numbers (PENs) are registered by IANA. It shows how to request a new PEN and how to modify a current PEN. It also gives a brief overview of PEN uses.

このドキュメントでは、プライベートエンタープライズ番号（PEN）がIANAによってどのように登録されているかについて説明します。新しいペンをリクエストする方法と、現在のペンを変更する方法を示しています。また、ペン用途の簡単な概要も提供します。

1. Introduction
1. はじめに

Private Enterprise Numbers (PENs) are identifiers that can be used anywhere that an ASN.1 object identifier (OID) [ASN1] can be used. Originally, PENs were developed so that organizations that needed to identify themselves in Simple Network Management Protocol (SNMP) [RFC3411] Management Information Base (MIB) configurations could do so easily. PENs are also useful in any application or configuration language that needs OIDs to identify organizations.

プライベートエンタープライズ番号（PEN）は、ASN.1オブジェクト識別子（OID）[ASN1]を使用できる場所で使用できる識別子です。もともと、ペンは、単純なネットワーク管理プロトコル（SNMP）[RFC3411]管理情報ベース（MIB）構成で自分自身を識別する必要がある組織が簡単に行うことができるように開発されました。ペンは、組織を識別するためにOIDを必要とするアプリケーションまたは構成言語でも役立ちます。

The IANA Functions Operator, referred to in this document as "IANA", manages and maintains the PEN registry in consultation with the IESG. PENs are issued from an OID prefix that was assigned to IANA. That OID prefix is 1.3.6.1.4.1. Using the (now archaic) notation of ownership names in the OID tree, that corresponds to:

このドキュメントで「IANA」と呼ばれるIANA機能演算子は、IESGと相談してPENレジストリを管理および維持しています。ペンは、IANAに割り当てられたOIDプレフィックスから発行されます。そのOIDプレフィックスは1.3.6.1.4.1です。OIDツリーの所有者名の（現在の古風な）表記を使用して、次のものに対応しています。

1 3 6 1 4 1 iso.org.dod.internet.private.enterprise

1 3 6 1 4 1 iso.org.dod.internet.private.enterprise

A PEN is an OID that begins with the PEN prefix. Thus, the OID 1.3.6.1.4.1.32473 is a PEN.

ペンは、ペンのプレフィックスから始まるOIDです。したがって、OID 1.3.6.1.4.1.32473はペンです。

1.1. Uses of PENs
1.1. ペンの用途

Once a PEN has been assigned to an organization, individual, or other entity, that assignee can use the PEN by itself (possibly to represent the assignee) or as the root of other OIDs associated with the assignee. For example, if an assignee is assigned the PEN 1.3.6.1.4.1.32473, it might use 1.3.6.1.4.1.32473.7 to identify a protocol extension and use 1.3.6.1.4.1.32473.12.3 to identify a set of algorithms that it supports in a protocol.

ペンが組織、個人、またはその他のエンティティに割り当てられたら、その譲受人はペンを単独で（おそらく譲受人を表すために）、または譲受人に関連付けられた他のOIDのルートとして使用できます。たとえば、譲受人がPEN 1.3.6.1.4.1.32473に割り当てられている場合、1.3.6.1.4.1.32473.7を使用してプロトコル拡張を特定し、1.3.6.1.4.1.32473.12.3を使用してアルゴリズムのセットを特定することができます。プロトコルでサポートすること。

Neither IANA nor the IETF can control how an assignee uses its PEN. In fact, no one can exert such control: that is the meaning of "private" in "private enterprise number". Similarly, no one can prevent an assignee that is not the registered owner of a PEN from using that PEN, or any PEN, however they want.

IANAもIETFも、譲受人がペンを使用する方法を制御できません。実際、誰もそのようなコントロールを発揮することはできません。それは「プライベートエンタープライズ番号」の「プライベート」の意味です。同様に、ペンの登録された所有者ではない譲受人がそのペンやペンを使用することを妨げることはできません。

A very common use of PENs is to give unique identifiers in IETF protocols. SNMP MIB configuration files use PENs for identifying the origin of values. Protocols that use PENs as identifiers of extension mechanisms include RADIUS [RFC2865], Diameter [RFC6733], Syslog [RFC5424], RSVP [RFC5284], and vCard [RFC6350].

ペンの非常に一般的な使用は、IETFプロトコルで一意の識別子を提供することです。SNMP MIB構成ファイルは、値の原点を識別するためにペンを使用します。ペンを拡張メカニズムの識別子として使用するプロトコルには、半径[RFC2865]、直径[RFC6733]、Syslog [RFC5424]、RSVP [RFC5284]、およびVCARD [RFC6350]が含まれます。

2. PEN Assignment
2. ペンの割り当て

PENs are assigned by IANA. The registry is located at <https://www.iana.org/assignments/enterprise-numbers>, and requests for new assignments or the modification of existing assignments can also be submitted at that URL.

ペンはIANAによって割り当てられます。レジストリは<https://www.iana.org/assignments/enterprise-numbers>にあり、新しい割り当てまたは既存の割り当ての変更のリクエストもそのURLで提出することができます。

IANA maintains the PEN registry in accordance with the "First Come First Served" registration policy described in [RFC8126]. Values are assigned sequentially.

IANAは、[RFC8126]に記載されている「First Come First First Serving」登録ポリシーに従ってPENレジストリを維持しています。値は順次割り当てられます。

2.1. Requesting a PEN Assignment
2.1. ペンの割り当てを要求します

Requests for assignment must provide the name of the assignee, the name of a public contact who can respond to questions about the assignment, and contact information that can be used to verify change requests. The contact's name and email address will be included in the public registry.

割り当てのリクエストは、譲受人の名前、割り当てに関する質問に回答できる公開連絡先の名前、および変更リクエストの検証に使用できる連絡先情報を提供する必要があります。連絡先の名前とメールアドレスは、パブリックレジストリに含まれます。

A prospective assignee may request multiple PENs, but obtaining one PEN and making internal sub-assignments is typically more appropriate. (Sub-assignments should not be reported to IANA.)

将来の譲受人は複数のペンを要求する場合がありますが、1つのペンを取得して内部サブ割り当てを行うことが通常適切です。（サブ割り当てはIANAに報告されるべきではありません。）

IANA may refuse to process abusive requests.

IANAは、虐待的な要求を処理することを拒否する場合があります。

2.2. Modifying an Existing Record
2.2. 既存のレコードの変更

Any of the information associated with a registered value can be modified, including the name of the assignee.

登録値に関連付けられている情報は、譲受人の名前を含めて変更できます。

Modification requests require authorization by a representative of the assignee. Authorization will be validated either with information kept on file with IANA or with other identifying documentation, if necessary.

修正要求には、譲受人の代表者による承認が必要です。許可は、IANAにファイルに保管されている情報または必要に応じて他の識別文書を使用して検証されます。

2.3. Deleting a PEN Record
2.3. ペンレコードの削除

Although such requests are rare, registrations can be deleted. When a registration is deleted, all identifying information is removed from the registry, and the value is marked as "returned." Returned values will not be made available for reassignment until all other unassigned values have been exhausted; as can be seen in Section 3, the unassigned values are unlikely to ever run out.

そのような要求はまれですが、登録は削除できます。登録が削除されると、すべての識別情報がレジストリから削除され、値は「返された」とマークされます。返された値は、他のすべての割り当てされていない値が使い果たされるまで、再割り当てのために利用できません。セクション3で見られるように、割り当てられていない値はこれまで尽きる可能性は低いです。

3. PEN Registry Specifics
3. ペンレジストリの詳細

The range for values after the PEN prefix is 0 to 2**32-1. The values 0 and 4294967295 (2**32-1) are reserved. Note that while the original PEN definition had no upper bound for the value after the PEN prefix, there is now an upper bound due to some IETF protocols limiting the size of that value. For example, Diameter [RFC6733] limits the value to 2**32-1.

ペンプレフィックス後の値の範囲は0〜2 ** 32-1です。値0および4294967295（2 ** 32-1）は予約されています。元のペン定義には、ペンプレフィックスの後に値の上限がありませんでしたが、その値のサイズを制限するIETFプロトコルのために上限があります。たとえば、直径[RFC6733]は値を2 ** 32-1に制限します。

There is a PEN number, 32473, reserved for use as an example in documentation. This reservation is described in [RFC5612].

ドキュメントの例として使用するために予約されているペン番号32473があります。この予約は[RFC5612]で説明されています。

Values in the registry that have unclear ownership are marked "Reserved". These values will not be reassigned to a new company or individual without consulting the IESG.

不明確な所有権を持つレジストリ内の値は、「予約済み」とマークされています。これらの値は、IESGに相談することなく、新しい会社または個人に再割り当てされることはありません。

このドキュメントによると、IANAは次の変更をペンレジストリに変更しました。

5. Security Considerations
5. セキュリティに関する考慮事項

Registering PENs does not introduce any significant security considerations.

ペンを登録しても、重要なセキュリティ上の考慮事項は導入されません。

There is no cryptographic binding of a registrant in the PEN registry and the PEN(s) assigned to them. Thus, the entries in the PEN registry cannot be used to validate the ownership of a PEN in use. For example, if the PEN 1.3.6.1.4.1.32473 is seen in a protocol as indicating the owner of some data, there is no way to securely correlate that use with the name and assignee of the owner listed in the PEN registry.

ペンレジストリに登録者の暗号化の結合はなく、ペンに割り当てられたペン。したがって、ペンレジストリのエントリを使用して、使用中のペンの所有権を検証することはできません。たとえば、PEN 1.3.6.1.4.1.32473がプロトコルにあるデータの所有者を示すものとして見られる場合、その使用をPENレジストリにリストした所有者の名前と譲受人と安全に相関させる方法はありません。
