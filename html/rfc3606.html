1. The Internet-Standard Management Framework
1. インターネット標準の管理フレームワーク

For a detailed overview of the documents that describe the current Internet-Standard Management Framework, please refer to section 7 of RFC 3410 [RFC3410].

現在のインターネット標準管理フレームワークを説明するドキュメントの詳細な概要については、RFC 3410 [RFC3410]のセクション7を参照してください。

Managed objects are accessed via a virtual information store, termed the Management Information Base or MIB. MIB objects are generally accessed through the Simple Network Management Protocol (SNMP). Objects in the MIB are defined using the mechanisms defined in the Structure of Management Information (SMI). This memo specifies a MIB module that is compliant to the SMIv2, which is described in STD 58, RFC 2578 [RFC2578], STD 58, RFC 2579 [RFC2579] and STD 58, RFC 2580 [RFC2580].

管理されたオブジェクトは、管理情報ベースまたはMIBと呼ばれる仮想情報ストアからアクセスされます。MIBオブジェクトは通常、単純なネットワーク管理プロトコル（SNMP）からアクセスされます。MIBのオブジェクトは、管理情報の構造（SMI）で定義されたメカニズムを使用して定義されます。このメモは、STD 58、RFC 2578 [RFC2578]、STD 58、RFC 2579 [RFC2579]およびSTD 58、RFC 2580 [RFC2580]に記載されているSMIV2に準拠したMIBモジュールを指定します。

2. Overview
2. 概要

The purpose of this memo is to provide additional capabilities, not found in the ATM-MIB [RFC2515], which are needed to manage ATM interfaces. This memo addresses the following areas:

このメモの目的は、ATM-MIB [RFC2515]にはない追加の機能を提供することです。これは、ATMインターフェイスを管理するために必要です。このメモは、次の領域に対応しています。

- ATM Switch Support - ATM Service Support - ATM Host Support

- ATMスイッチサポート - ATMサービスサポート-ATMホストサポート

In addition, this memo also provides ATM trap support.

さらに、このメモはATMトラップサポートも提供します。

2.1. Background
2.1. 背景

In addition to the MIB module defined in this memo, other MIB modules are necessary to manage ATM interfaces, links and cross-connects. Examples include MIB II for general system and interface management ([RFC2863]), the DS3 ([RFC2496]) or SONET MIBs ([RFC3592]) for management of SONET and DS3 physical interfaces, and, as appropriate, MIB modules for applications that make use of ATM, such as SMDS [RFC1694] and LAN Emulation [ATM Forum LANE]. These MIB modules are outside the scope of this specification.

このメモで定義されているMIBモジュールに加えて、ATMインターフェイス、リンク、およびクロスコネクトを管理するために、他のMIBモジュールが必要です。例には、一般的なシステムとインターフェイス管理のためのMIB II（[RFC2863]）、DS3（[RFC2496））またはSONETおよびDS3物理インターフェイスの管理のためのSONET MIBS（[RFC3592]）、および適切な場合、アプリケーションのMIBモジュールSMDS [RFC1694]やLANエミュレーション[ATMフォーラムレーン]などのATMを使用します。これらのMIBモジュールは、この仕様の範囲外です。

This MIB module also requires the use of the ATM-MIB module defined in [RFC2515] and ATM-specific textual conventions defined in [RFC2514].

このMIBモジュールには、[RFC2515]で定義されたATM-MIBモジュールと[RFC2514]で定義されたATM固有のテキスト規則の使用も必要です。

ATM Endpoint applications such as ATM LAN Emulation or Classical IP-over-ATM Clients and Servers use ATM to establish SVC/PVC connections for exchanging control and data information. The agents of these ATM applications must provide the network manager with information on the SVC/PVCs in use and which applications are using them. The information can be made generic so as to apply to all ATM applications. This memo defines extensions to the ATM-MIB [RFC2515] in order to support this.

ATM LANエミュレーションやクラシックIPオーバーATMクライアントやサーバーなどのATMエンドポイントアプリケーションは、ATMを使用して、制御情報とデータ情報を交換するためにSVC/PVC接続を確立します。これらのATMアプリケーションのエージェントは、使用中のSVC/PVCおよびそれらを使用しているアプリケーションに関する情報をネットワークマネージャーに提供する必要があります。情報は、すべてのATMアプリケーションに適用するために一般的にすることができます。このメモは、これをサポートするためにATM-MIB [RFC2515]への拡張機能を定義します。

The current specification of this supplemental ATM2-MIB is based on SNMPv2 SMI.

この補足ATM2-MIBの現在の仕様は、SNMPV2 SMIに基づいています。

3.4. ATM-related Trap Support
3.4. ATM関連のトラップサポート

Traps are defined to detect changes in the status of permanent VPLs and VCLs. The current up/down status of each permanent VPL or VCL is indicated by the atmVplOperStatus or atmVclOperStatus object, respectively. Several tables and objects and one trap are defined in order to help network managers quickly and efficiently detect changes in the status of permanent virtual links. Through use of these tables, objects, and traps, the time consuming and resource intensive task of continuously polling each row in the entire atmVplTable and atmVclTable can be avoided.

トラップは、永続的なVPLとVCLのステータスの変化を検出するために定義されます。各永久VPLまたはVCLの現在のアップ/ダウンステータスは、それぞれATMVPloperstatusまたはAtmVCloperstatusオブジェクトによって示されています。ネットワークマネージャーが永続的な仮想リンクのステータスの変更を迅速かつ効率的に検出できるように、いくつかのテーブルとオブジェクトと1つのトラップが定義されています。これらのテーブル、オブジェクト、トラップを使用することにより、ATMVPLTABLE全体で各行を継続的にポーリングする時間をかける時間とリソースの集中的なタスクを回避できます。

The atmIntfPvcFailures counter and the atmIntfCurrentlyFailingPVpls and atmIntfCurrentlyFailingPVcls gauges provide a quick means of determining the status of all PVPLs and PVCLs on an interface. The atmCurrentlyFailingPVplTable and the atmCurrentlyFailingPVclTable list all of the problematic PVPLs and PVCLs, respectively, allowing them to be quickly identified.

ATMINTFPVCFAILURESカウンターおよびATMINTFCURLENTSFAILINGPVPLSおよびATMINTFCURRENTSFAILINGPVCLSゲージは、インターフェイス上のすべてのPVPLSおよびPVCLのステータスを決定する迅速な手段を提供します。ATMCURLENTSFAILINGPVPLTABLEとATMCURLENTSFAILINGPVCLTABLEは、それぞれすべての問題のあるPVPLとPVCLをリストし、それらを迅速に識別できるようにします。

The atmIntfPvcFailuresTrap is generated just after a PVPL or PVCL on a particular interface leaves the 'up' operational state. Managers can then determine which PVPLs and/or PVCLs are failing by reading the atmCurrentlyFailingPVplTable and the atmCurrentlyFailingPVclTable. Generation of the atmIntfPvcFailuresTrap is rate limited by suppressing all traps that would occur within atmIntfPvcNotificationInterval of a previous trap for the same interface. Managers should continuously poll the tables and objects mentioned above for at least this amount of time in order to keep up with the state of the network.

ATMINTFPVCFAILURESTRAPは、特定のインターフェイス上のPVPLまたはPVCLが「UP」動作状態を離れる直後に生成されます。次に、マネージャーは、ATMCURLENTSFAILINGPVPLTABLEとATMCURLENTSFAILINGPVCLTABLEを読み取ることにより、どのPVPLおよび/またはPVCLが失敗しているかを決定できます。ATMINTFPVCFAILURESTRAPの生成は、同じインターフェイスの前のトラップのATMINTFPVCNOTIFIITIONVELVAL内で発生するすべてのトラップを抑制することにより、レートに制限されます。マネージャーは、ネットワークの状態に追いつくために、少なくともこの時間は上記のテーブルとオブジェクトを継続的に投票する必要があります。
