New ASN.1 Modules for the Evidence Record Syntax (ERS)

新しいASN.1証拠記録構文（ERS）のモジュール

Abstract

概要

The Evidence Record Syntax (ERS) and the conventions for including these evidence records in the Server-based Certificate Validation Protocol (SCVP) are expressed using ASN.1. This document offers alternative ASN.1 modules that conform to the 2002 version of ASN.1 and employ the conventions adopted in RFCs 5911, 5912, and 6268. There are no bits-on-the-wire changes to any of the formats; this is simply a change to the ASN.1 syntax.

証拠記録構文（ERS）とサーバーベースの証明書検証プロトコル（SCVP）のこれらの証拠記録を含めるための規則は、ASN.1を使用して表現されます。この文書は、2002年のASN.1に準拠し、RFC 5911,5912、および6268に採用されている規約を採用している代替ASN.1モジュールを提供しています。これは単にASN.1構文への変更です。

1. Introduction
1. はじめに

Some developers would like the IETF to use the latest version of ASN.1 in its standards. This document provides alternative ASN.1 modules to assist in that goal.

一部の開発者は、IETFがその標準内の最新バージョンのASN.1を使用したいと思います。この文書は、その目標を支援するための代替ASN.1モジュールを提供します。

The Evidence Record Syntax (ERS) [RFC4998] provides two ASN.1 modules: one using the 1988 syntax [OLD-ASN1], which has been deprecated by the ITU-T, and another one using the newer syntax [NEW-ASN1], which continues to be maintained and enhanced. This document provides an alternative ASN.1 module that follows the conventions established in [RFC5911], [RFC5912], and [RFC6268].

証拠レコード構文（ERS）[RFC4998]は、2つのASN.1モジュールを提供します.1つは、ITU-Tによって廃止予定されています。これは維持され強化され続けます。この資料は、[RFC5911]、[RFC5912]、[RFC6268]で確立された規則に続く代替ASN.1モジュールを提供します。

In addition, [RFC5276] specifies the mechanism for conveying evidence records in the Server-based Certificate Validation Protocol (SCVP) [RFC5055]. There is only one ASN.1 module in [RFC5276], and it uses the 1988 syntax [OLD-ASN1]. This document provides an alternative ASN.1 module using the newer syntax [NEW-ASN1] and follows the conventions established in [RFC5911], [RFC5912], and [RFC6268]. Note that [RFC5912] already includes an alternative ASN.1 module for SCVP [RFC5055].

また、[RFC5276]サーバーベースの証明書検証プロトコル（SCVP）[RFC5055]の証拠記録を伝達するためのメカニズムを指定します。[RFC5276]には1つのASN.1モジュールが1つしかありません。これは1988年の構文[old-asn1]を使用します。この資料は、新しい構文[New-ASN1]を使用した代替ASN.1モジュールを提供し、[RFC5911]、[RFC5912]、[RFC6268]で確立された規則に従います。[RFC5912]はすでにSCVPの代替ASN.1モジュールを含んでいます[RFC5055]。

The original ASN.1 modules get some of their definitions from places outside the RFC series. Some of the referenced definitions are somewhat difficult to find. The alternative ASN.1 modules offered in this document stand on their own when combined with the modules in [RFC5911], [RFC5912], and [RFC6268].

元のASN.1モジュールは、RFCシリーズの外部からの定義のいくつかを取得します。参照されている定義のいくつかは、見つけるのが多少難しいです。このドキュメントで提供される代替ASN.1モジュールは、[RFC5911]、[RFC5912]、[RFC6268]のモジュールと組み合わせると独自のものです。

The alternative ASN.1 modules produce the same bits on the wire as the original ones.

代替のASN.1モジュールは、元のものとしてワイヤー上の同じビットを生成します。

The alternative ASN.1 modules are informative; the original ones are normative.

代替ASN.1モジュールは有益です。元のものは規範的です。

        FROM PKIX-CommonTypes-2009 -- in [RFC5912]

      ;
        
      ltans OBJECT IDENTIFIER ::= { iso(1) identified-organization(3)

      EvidenceRecord ::= SEQUENCE {
        version  INTEGER { v1(1) },
        digestAlgorithms  SEQUENCE OF AlgorithmIdentifier
                            {DIGEST-ALGORITHM, {...}},
        cryptoInfos  [0] CryptoInfos OPTIONAL,
        encryptionInfo  [1] EncryptionInfo OPTIONAL,
        archiveTimeStampSequence  ArchiveTimeStampSequence }
        
      CryptoInfos ::= SEQUENCE SIZE (1..MAX) OF Attribute
        
      ArchiveTimeStamp ::= SEQUENCE {
        digestAlgorithm [0] AlgorithmIdentifier
                              {DIGEST-ALGORITHM, {...}} OPTIONAL,
        attributes      [1] Attributes OPTIONAL,
        reducedHashtree [2] SEQUENCE OF PartialHashtree OPTIONAL,
        timeStamp       ContentInfo }
        
      PartialHashtree ::= SEQUENCE OF OCTET STRING
        
      Attributes ::= SET SIZE (1..MAX) OF Attribute
        
      ArchiveTimeStampChain ::= SEQUENCE OF ArchiveTimeStamp
        
      ArchiveTimeStampSequence ::= SEQUENCE OF ArchiveTimeStampChain
        
      EncryptionInfo ::= SEQUENCE {
        encryptionInfoType  ENCINFO-TYPE.&id
          ({SupportedEncryptionAlgorithms}),
        encryptionInfoValue  ENCINFO-TYPE.&Type
          ({SupportedEncryptionAlgorithms}{@encryptionInfoType}) }
        
      ENCINFO-TYPE ::= TYPE-IDENTIFIER
        
      SupportedEncryptionAlgorithms ENCINFO-TYPE ::= { ... }
        
      aa-er-internal ATTRIBUTE ::=
        { TYPE EvidenceRecord IDENTIFIED BY id-aa-er-internal }
        
      id-aa-er-internal  OBJECT IDENTIFIER ::= { iso(1) member-body(2)
        us(840) rsadsi(113549) pkcs(1) pkcs9(9) smime(16) id-aa(2) 49 }
        
      aa-er-external ATTRIBUTE ::=
        { TYPE EvidenceRecord IDENTIFIED BY id-aa-er-external }
