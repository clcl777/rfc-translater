Assignment of an Ethertype for IPv6 with Low-Power Wireless Personal Area Network (LoWPAN) Encapsulation

低電力ワイヤレスパーソナルエリアネットワーク（LoWPAN）カプセル化によるIPv6のEthertypeの割り当て

Abstract

概要

When carried over Layer 2 technologies such as Ethernet, IPv6 datagrams using Low-Power Wireless Personal Area Network (LoWPAN) encapsulation as defined in RFC 4944 must be identified so the receiver can correctly interpret the encoded IPv6 datagram. The IETF officially requested the assignment of an Ethertype for that purpose and this document reports that assignment.

イーサネットなどのレイヤー2テクノロジで伝送される場合、RFC 4944で定義されている低電力ワイヤレスパーソナルエリアネットワーク（LoWPAN）カプセル化を使用するIPv6データグラムを識別して、受信者がエンコードされたIPv6データグラムを正しく解釈できるようにする必要があります。 IETFはその目的のためにEthertypeの割り当てを公式に要求し、このドキュメントはその割り当てを報告します。

1. Introduction
1. はじめに

The IETF defined "LoWPAN encapsulation", a format for IPv6 [RFC2460] datagram encapsulation [RFC4944]. This document regards any IPv6 datagram using the Dispatch octet as defined in Section 5.1 of RFC 4944 as using LoWPAN encapsulation. LoWPAN encapsulation as defined in RFC 4944 has been updated by [RFC6282], and may be extended and modified by future IETF Standards. The intended Layer 2 technology for IPv6 datagrams using LoWPAN encapsulation as originally defined is [IEEE.802.15.4_2011], which does not provide for a protocol switch in its Layer 2 headers.

IETFは、IPv6 [RFC2460]データグラムカプセル化[RFC4944]の形式である「LoWPANカプセル化」を定義しました。このドキュメントでは、RFC 4944のセクション5.1で定義されているDispatchオクテットを使用するIPv6データグラムを、LoWPANカプセル化を使用するものと見なしています。 RFC 4944で定義されているLoWPANカプセル化は[RFC6282]によって更新されており、将来のIETF標準によって拡張および変更される可能性があります。当初定義されたLoWPANカプセル化を使用するIPv6データグラム用のレイヤー2テクノロジーは、[IEEE.802.15.4_2011]であり、レイヤー2ヘッダーでプロトコルスイッチを提供していません。

The following list gives some example use cases for the Ethertype-based protocol dispatch for the LoWPAN-encapsulated IPv6 datagram:

次のリストは、LoWPANカプセル化IPv6データグラムのEthertypeベースのプロトコルディスパッチのいくつかの使用例を示しています。

o Usage of LoWPAN encapsulation in conjunction with IEEE 802.15.9 Multiplexed Data Service [IEEE802159], which provides the ability to perform upper layer protocol dispatch for IEEE 802.15.4 networks. Wi-SUN Alliance intends to use the 15.9 Multiplexed Data Information Element for upper layer protocol dispatch. As specified in IEEE 802.15.9, dispatch of LoWPAN encapsulation frames will require that an Ethertype be assigned for LoWPAN encapsulation.

o IEEE 802.15.9多重化データサービス[IEEE802159]と組み合わせたLoWPANカプセル化の使用。IEEE802.15.4ネットワークに上位層プロトコルディスパッチを実行する機能を提供します。 Wi-SUN Allianceは、上位層プロトコルのディスパッチに15.9多重化データ情報要素を使用する予定です。 IEEE 802.15.9で指定されているように、LoWPANカプセル化フレームのディスパッチでは、LoWPANカプセル化にEthertypeを割り当てる必要があります。

o LoWPAN encapsulation will likely be needed for Wi-Fi Alliance's HaLoW [HALOW] standard (low-power operation in the 900 MHz band).

o LoWPANカプセル化は、Wi-Fi AllianceのHaLoW [HALOW]標準（900 MHz帯域での低電力動作）で必要になる可能性があります。

o Other Layer 2 technologies such as Ethernet and debugging tools such as Wireshark require a unique Protocol Type field for LoWPAN encapsulation to properly interpret IPv6 datagrams that use LoWPAN encapsulation.

o イーサネットなどの他のレイヤー2テクノロジーやWiresharkなどのデバッグツールでは、LoWPANカプセル化を使用するIPv6データグラムを正しく解釈するには、LoWPANカプセル化に固有のプロトコルタイプフィールドが必要です。

o Any existing or future Layer 2 technology, incorporating Ethertype-based upper layer dispatch, can use the Ethertype proposed in this document to dispatch LoWPAN-encapsulated IPv6 datagrams.

o Ethertypeベースの上位層ディスパッチを組み込んだ既存または将来のレイヤ2テクノロジーは、このドキュメントで提案されているEthertypeを使用して、LoWPANカプセル化IPv6データグラムをディスパッチできます。

2. Assignment of an Ethertype by IEEE
2. IEEEによるEthertypeの割り当て

The IETF formally submitted a request to IEEE for assignment of an Ethertype for IPv6 datagrams using LoWPAN encapsulation. IEEE has responded with the assignment of Ethertype 0xA0ED for this purpose. The assignment has been recorded by IEEE [IEEE-ETHERTYPES].

IETFは、LoWPANカプセル化を使用してIPv6データグラムのEthertypeを割り当てる要求を正式にIEEEに送信しました。 IEEEは、この目的のためにEthertype 0xA0EDの割り当てで応答しました。割り当てはIEEE [IEEE-ETHERTYPES]によって記録されています。

4. Security Considerations
4. セキュリティに関する考慮事項

This document only requests assignment of an Ethertype for IPv6 datagrams using LoWPAN encapsulation. It has no incremental implications for security beyond those in the relevant protocols.

このドキュメントでは、LoWPANカプセル化を使用したIPv6データグラムのEthertypeの割り当てのみを要求しています。これは、関連するプロトコルのセキュリティを超えるセキュリティへの影響はありません。
